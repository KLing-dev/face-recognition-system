# 人脸识别系统
基于Python + Flask + Vue3 + MTCNN + FaceNet的前后端分离人脸识别系统，支持人脸数据录入、实时/照片识别、人数统计等功能。

## 依赖安装
### 后端依赖（Python 3.11）
1. 激活conda虚拟环境：`conda activate face_recog`
2. 进入backend目录：`cd backend`
3. 安装依赖：`pip install -r requirements.txt`

### 前端依赖（Node.js 16+）
1. 安装Node.js（官网：https://nodejs.org/ ，选择16.x版本）
2. 进入frontend目录：`cd frontend`
3. 安装依赖：`npm install`

## 系统运行
### 1. 启动后端

## Git工作流规范
### 分支管理
- `main`：主分支，存放生产环境代码，仅通过合并`dev`分支更新
- `dev`：开发分支，存放测试通过的代码，由`feature`分支合并而来
- `feature/xxx`：功能分支，如`feature/user-register`（用户录入功能），开发完成后合并到`dev`

### 提交规范
- 格式：`type(scope): description`
- 示例：
  - `feat(backend): 实现人脸特征提取功能`
  - `fix(frontend): 修复图片上传失败问题`
  - `docs: 更新README.md接口文档`
- type类型：feat（新功能）、fix（修复）、docs（文档）、style（样式）、refactor（重构）、test（测试）、chore（构建/依赖）

### 合并流程
1. 功能开发完成后，提交`feature`分支到远程仓库
2. 创建Pull Request（PR）到`dev`分支
3. 代码审核通过后，合并到`dev`分支
4. 测试环境验证通过后，由`dev`分支合并到`main`分支

## 核心算法模块说明
### 1. 人脸检测（MTCNN）
- 功能：从图片中检测人脸位置（ bounding box ），过滤非人脸区域
- 优势：轻量、高效，支持多人脸检测
- 流程：图片输入 → MTCNN模型 → 输出人脸坐标和置信度（过滤低置信度结果）
- 参数优化：
  - `min_face_size`：最小可检测人脸大小（默认15）
  - `steps_threshold`：三级网络的阈值（默认[0.6, 0.7, 0.75]，已优化）
  - `scale_factor`：图像缩放因子（默认0.7，已优化）
  - `target_region`：目标区域优先级（可选，用于优化人脸区域识别）

### 2. 人脸特征提取（FaceNet）
- 功能：将检测到的人脸图像转换为128维的特征向量
- 优势：特征向量具有良好的区分性，支持相似度计算
- 流程：人脸图像 → 预处理（归一化、对齐） → FaceNet预训练模型 → 128维特征向量
- 优化：增加了图像预处理增强、人脸对齐和特征归一化

### 3. 特征比对（余弦相似度）
- 功能：计算输入人脸特征与数据库中所有特征的余弦相似度
- 判定规则：相似度 > 阈值（0.55） → 匹配成功，返回对应用户名
- 公式：cos_sim(a,b) = (a·b) / (||a||×||b||)
- 优化：增加了接近阈值的加权逻辑，提高识别准确性

### 4. 人脸区域手工调整机制
- 功能：允许用户在自动识别后对人脸区域进行手工二次调整
- 主要组件：
  - `FaceAdjustment`类：提供图像预处理、人脸区域自动检测、手工调整、验证等功能
  - 支持区域扩展、微调、撤销等操作

### 5. 用户编号自动生成机制
- 功能：自动生成唯一的用户编号
- 编号规则：USR + 年月日 + 4位序号
- 主要组件：
  - `UserIDGenerator`类：提供编号生成、验证、唯一性检查等功能
  - 支持自定义前缀、备选生成方案

### 6. 人脸唯一性校验与提示
- 功能：校验注册人脸的唯一性，防止重复注册
- 校验逻辑：
  - 与数据库中所有已注册人脸进行特征比对
  - 根据相似度级别（极高、高、中等、低、极低）返回不同的提示信息
  - 提供二次确认流程
- 主要组件：
  - `FaceUniquenessChecker`类：提供特征提取、唯一性校验、提示生成等功能

### 7. 用户数据删除功能优化
- 功能：提供安全、灵活的用户数据删除机制
- 主要功能：
  - 支持通过用户ID或用户名删除用户
  - 支持单个或多个用户数据删除
  - 提供删除确认机制，防止误操作
  - 同时删除数据库记录和关联的图像文件
- 主要组件：
  - `UserDataManager`类：提供用户查询、删除、确认等功能
  - 支持模拟删除、数据库完整性检查等高级功能

## 故障排除
### 1. 环境安装失败
- 问题：MTCNN/FaceNet安装失败 → 解决方案：确保Python版本为3.11，先安装依赖`pip install setuptools wheel`，再重新安装
- 问题：OpenCV导入失败 → 解决方案：使用`pip install opencv-python-headless`（无GUI版本）

### 2. 人脸识别准确率低
- 解决方案：调整`RECOGNITION_THRESHOLD`阈值（0.5-0.7之间），确保录入人脸时光线充足、正面拍摄

### 3. 前端无法访问后端接口
- 解决方案：检查后端是否启动（http://127.0.0.1:5000），确认后端已安装`flask-cors`（跨域支持）

### 4. 摄像头无法调用
- 解决方案：浏览器授权摄像头访问，确保前端代码中摄像头权限申请正确，后端接口支持base64图片传输

### 5. 人脸区域识别错误
- 解决方案：已通过优化MTCNN参数（调整steps_threshold=[0.6,0.7,0.75]、scale_factor=0.7）和新增综合评分排序机制修复

### 6. 重复人脸注册
- 解决方案：使用新增的人脸唯一性校验机制，设置UNIQUENESS_THRESHOLD=0.5，可有效防止重复注册

### 7. 误删除用户数据
- 解决方案：使用用户数据删除功能的确认机制，系统会自动提示确认，防止误操作

### 8. 编号冲突
- 解决方案：用户编号自动生成机制采用年月日+序号的方式，并通过数据库查询确保唯一性
